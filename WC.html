AtCoder Educational DP Contest T - Permutation

题目地址

  • https://atcoder.jp/contests/dp/tasks/dp_t

描述

  • 有一个长为N的正整数序列。给定一个由'<'和'>'组成长为N-1的的字符串。
  • 对于任意满足1≤i≤N−1的字符si,如果si是'<'则Pi<Pi+1、如果si是'>'则Pi>Pi+1。求满足这样的性质的排列P的方案数。

解法

  • 设 f[i][j] 表示已经考虑了前 i 个位置，且第 i 位上，填的数字是第 j小的总方案数。

    不难想到初始状态即用 1 填了 1 位，即 f[1][1] = 1 。

    由于每一位必由前一个位置转移过来，所以可以分两种情况：

    • 这个位置是 <，根据状态定义，前一位要比这个小，即 f[i][j] += f[i - 1][k] (k < j)；
    • 这个位置是 >，前一位要比这个大，但是，在i−1 时的第 j 小，由于现在加入了一个第 j 小的数，将上次的第 j 小变成了现在的 j+1 小，所以需要加上 f[i - 1][k]。即 f[i][j] += f[i - 1][k] (j ≤ k < i)；

代码

#include <iostream>
#include <stdio.h>


#define int long long
#define maxn 3005
#define Mod 1000000007
int n;
char s[maxn] ;
int f[maxn][maxn] ;
int t[maxn][maxn] ;
using namespace std ;


int readInt () ;


int getsum (int x , int sy , int ey) {
    
    return (t[x][ey] - t[x][sy - 1]) % Mod ;
}


void sumPrefix (int x ) {
    t[x][0] = 0 ;
    for (int i = 1 ; i <= n ; ++ i) {
        t[x][i] = (t[x][i - 1] + f[x][i]) % Mod ;
    }
}
signed main () {
//    freopen ("data.txt" , "r" , stdin) ;
    n = readInt () ;
    scanf ("%s" , s) ;
    
    //f[i][j]为前i个符合条件，且最后一个为第j大的数。
    
    
    f[1][1] = 1 ;
    for (int i = 2 ; i <= n ; ++ i) {
        
        sumPrefix(i - 1) ;
        for (int j = 1 ; j <= i ; ++ j) {
            
            if (s[i - 2] == '>') {
                f[i][j] = getsum (i - 1, 1 , j - 1) ;
            } else {
                f[i][j] = getsum (i - 1 ,j,  n) ;
            }
            
//            cout << i <<' ' << j <<' ' << f[i][j] << endl;
        }
        
    }
    
    int ans = 0 ;
    for (int i = 1 ; i <= n ; ++ i) {
        ans += f[n][i] ;
        ans %= Mod ;
    }
    
    ans %= Mod ;
    ans += Mod ;
    ans %= Mod ;
    
    cout << ans << endl;
    return 0 ;
}

sequence

描述

  • 有多少个1…n的排列，恰好有k对相邻的数满足前面的比后面的数大？
  • 其中0≤k<n≤500

解法

方案1

  • 设f[i][j][k]表示前i个位置，第i个位置是到目前为止第j大的，满足条件的数恰好有k对的方案数的个数。

  • f[i+1][j][k]=∑t=1j−1f[i][t][k]+∑t=jif[i][t][k−1]+1

  • 其复杂度为O(n2k)

方案2

  • 设f[i][j][k]表示考虑了1…i这些数，这i个数在最后的排列中构成了连续的j段，到目前为止满足条件的数对恰好有k对的方案数。

[Atcoder Beginner Contest 282G] Similar Permutation

描述

  • 有长度为N的排列A和B。
  • 定义序列的相似性为满足(Ai+1−Ai)(Bi+1−Bi)>0的数量
  • 求相似度为K的A和B的数量。
  • 2≤N≤100

解法

  • f[i][j][k][l]表示前i个位置，A的第i个位置是到目前位置第j大的，B的第i个位置是到目前第k大的，满足条件的恰好有l对的方案有多少个。

  • f[i][j][k][l]=∑x=1,y=1x=i,y=if[i−1][x][y][l−1]∗[x−i>0]∗[y−j>0]
